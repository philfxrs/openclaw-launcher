# OpenClaw Windows 安装器本地测试说明

## 目标

本地测试的目标不是验证“作者电脑能跑”，而是验证以下分发级场景：

- 干净 Windows 10/11 x64 环境可完成首次安装
- 缺失依赖可被自动补齐
- OpenClaw 可被完整安装并完成首次启动
- 桌面快捷方式可正常启动
- 重复安装 / 修复安装行为可预测
- 失败时日志、错误码、状态文件足够定位问题
- 中文界面、中文错误提示、中文日志无乱码

## 测试环境

### 1. 当前开发机

用于验证：

- 资源同步脚本
- 启动器编译
- Inno Setup 打包
- 安装脚本静态检查
- 现有环境下的重复安装 / 修复逻辑

### 2. 干净 Windows 虚拟机 / Windows Sandbox

用于验证：

- 真实用户首次安装路径
- Node.js / Git / WebView2 自动补装
- 安装后自动启动
- 桌面快捷方式可用
- 中文显示是否正常

### 3. 部分依赖已存在的机器

用于验证：

- 依赖识别是否准确
- 是否跳过不必要安装
- 是否仍能顺利完成 OpenClaw 安装

### 4. 重复安装 / 修复测试环境

用于验证：

- 已安装 OpenClaw 时再次执行安装包
- 中途失败后再次运行安装包
- 快捷方式缺失后的补建

## 执行顺序

1. 在当前开发机运行静态检查与本地构建。
2. 在当前开发机执行一次安装与卸载链路验证。
3. 在干净 Windows 虚拟机 / Sandbox 走完整首次安装。
4. 在部分依赖已存在环境执行补装验证。
5. 在失败恢复 / 重复安装环境执行修复验证。
6. 只有全部通过后，才允许整理 GitHub Release 产物与说明。

## 测试用例

### T001 全新环境首次安装

- 前提：全新 Windows 10/11 x64，无 Node.js / Git / OpenClaw
- 步骤：运行 OpenClawSetup.exe，按默认流程完成安装
- 预期：依赖自动安装、OpenClaw 安装成功、桌面图标创建成功、安装结束后自动启动
- 失败判定：任一步骤报错、缺依赖未补齐、未自动启动、桌面图标不存在

### T002 缺 Node.js 环境安装

- 前提：无 Node.js，其他依赖可存在或不存在
- 步骤：运行安装器并观察 Node.js 检测与安装日志
- 预期：检测到 Node.js 缺失，自动安装 Node.js，验证通过
- 失败判定：Node.js 安装后 `node -v` 不可用或版本低于要求

### T003 缺 Git 环境安装

- 前提：无 Git for Windows
- 步骤：运行安装器并观察 Git 检测与安装日志
- 预期：自动安装 Git，验证后 `git --version` 可用
- 失败判定：Git 安装未执行、执行失败或 PATH 不可用

### T004 部分依赖已存在安装

- 前提：已安装 Node.js 或 Git 中的一部分，未安装 OpenClaw
- 步骤：运行安装器
- 预期：已存在依赖被识别并跳过，缺失依赖被补装，最终安装成功
- 失败判定：重复安装已有依赖导致异常，或缺失依赖未补齐

### T005 OpenClaw 已安装再次执行安装

- 前提：机器上已有健康 OpenClaw 安装
- 步骤：再次运行安装器
- 预期：按 ExistingInstallAction 的默认 Auto 策略复用健康安装，仅修复缺失组件
- 失败判定：破坏现有可用安装、重复安装导致失败或无法启动

### T006 安装中断后再次安装

- 前提：第一次安装在中途被人为中止或因网络/依赖失败中断
- 步骤：再次运行安装器
- 预期：状态文件保留，已完成步骤不做破坏性回滚，可继续执行或安全修复
- 失败判定：状态文件损坏、重复执行导致环境损坏、无法继续安装

### T007 安装完成后桌面图标启动

- 前提：安装已成功完成
- 步骤：双击桌面 OpenClaw 图标
- 预期：启动器进程出现，OpenClaw 界面可见
- 失败判定：快捷方式不存在、目标无效、双击后无窗口

### T008 安装日志输出验证

- 前提：执行一轮完整安装
- 步骤：检查 `%ProgramData%\OpenClawInstaller\Logs\`
- 预期：存在 `.log`、`.jsonl`、状态文件，日志含步骤编号、命令、退出码、错误码
- 失败判定：日志缺失、关键字段缺失、无法定位失败步骤

### T009 中文界面与日志无乱码验证

- 前提：运行安装器，并至少产生一条中文 UI / 日志内容
- 步骤：观察安装详情页、失败提示、日志文件
- 预期：中文均正常显示，无乱码、无问号、无 mojibake
- 失败判定：任一界面或日志出现乱码

### T010 自动启动验证

- 前提：安装成功且未传入 SkipLaunchAfterInstall
- 步骤：安装结束后观察是否自动启动
- 预期：Launcher 进程出现，验证脚本通过
- 失败判定：安装成功但未自动启动，或启动后立即退出

### T011 快捷方式目标有效性验证

- 前提：安装成功
- 步骤：检查桌面与开始菜单快捷方式的 TargetPath
- 预期：目标均指向 `OpenClawLauncher.exe`
- 失败判定：目标缺失、指向错误位置、文件不存在

### T012 卸载后重新安装验证

- 前提：已有安装
- 步骤：通过系统卸载入口卸载，然后重新运行安装器
- 预期：卸载清理完成，重新安装后仍可正常启动
- 失败判定：卸载残留阻塞重装、重装后启动失败、快捷方式异常

## 发布前通过标准

只有以下条件全部满足，才允许进入 GitHub 发布准备：

- 安装器可在本机成功构建
- 核心安装流程通过
- 依赖自动安装通过
- OpenClaw 安装通过
- 自动启动通过
- 桌面快捷方式通过
- 日志完整并包含错误码
- 中文界面与日志无乱码
- 至少一轮干净系统测试通过
- 至少一轮重复安装 / 修复测试通过

## 执行命令

建议先运行：

```powershell
powershell -ExecutionPolicy Bypass -File .\build\Test-LocalInstaller.ps1 -RepositoryRoot .
```

然后在当前电脑上执行兼容性矩阵测试，模拟不同依赖环境：

```powershell
powershell -ExecutionPolicy Bypass -File .\build\Test-LocalCompatibilityMatrix.ps1 -RepositoryRoot .
```

该脚本会在不卸载本机真实依赖的前提下，模拟：

- 全新机器
- 缺 Node.js
- 缺 Git
- 缺 WebView2
- 部分依赖已存在

然后启动 Windows Sandbox 验证干净环境路径：

```powershell
powershell -ExecutionPolicy Bypass -File .\build\Start-WindowsSandbox-InstallerTest.ps1 -RepositoryRoot .
```

Sandbox 启动后会自动：

- 打开安装器所在共享目录
- 打开测试计划
- 打开测试记录模板
- 启动 `OpenClawSetup.exe`

测试记录请填写：

- `installer/docs/local-test-record-template.md`

然后在虚拟机 / Sandbox 中执行手工验收用例 T001-T012。
